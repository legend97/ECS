A Valid HTML Page

Before any code can actually be loaded, the <!DOCTYPE html> declaration must be used to let the system know that the following code is HTML. The next necessary thing is to let the code know the location of the html code. This is done by inserting the <html> tag. The optional <head> element can be used to store metadata, which provides the data necessary for the data that is shown in the HTML document. The metadata is not shown, but it defines the data in terms of character set, title, styles, links, scripts, and many more. After the <head>, the actual content for the HTML code must be inserted. The <body> tag assigns a place for all of the content in the page. However, you cannot simply plug in all of the information that will be used in the <body> tag. You can apply some headers or paragraphs (<head><p>) in order to separate some of the content. After the line is finished, you need to include the end tag. The end tag is the same as the start tag, except there is a slash. Some of the necessary end tags include: </html>, </head>, </body>, </h1>, and </body>.

HTML

<html> is a tag that defines the ENTIRE document, including all the tags inside of it. It tells the code where the html code is, and is located after the initial HTML declaration in the beginning.

Headers

The headers tags go at the beginning of the code, right after the <html> tag. The header contains all the metadata for the entire HTML page. Metadata is data about the data. It summarizes all the basic data for the data that is introduced later on, such as character set, title, and many other things. The tags that can go into the <head> tag are: <title>, <style>, <base>, <link>, <meta>, <script>, and <noscript>

Examples

The example below will make title of this webpage "Headers" with the <title> element.

<head>
<title>headers</title>
</head>

Comments

Links

Links, which are pretty self-explanatory, are elements that link to internal or external sources. The links are defined with the <a> tag. There is the "href" element after the <a but before the end bracket, indicating that it is a hyperlink to another source. After the link has been inserted, the name of the link must be put in as well.

Examples

<a href="google.com">Google</a>
The code above will show the link to Google.com, but it will say "Google" as the link's name.

Lists

HTML can display lists of objects, either through an ordered list or an unordered list. Ordered lists have some sort of organization that always shows the correct order. Objects in an unordered list do not have a particular spot, and it can go wherever it wants to. The ordered lists are shown through the <ol> tag, and unordered lists are shown through the <ul> tag. For both ordered and unordered lists, there has to be a way of notifying the system that an object is part of the list. Therefore, the <li> must always precede the element that is being considered as a list.

Examples

Unordered List:

  • Math
  • Science
  • Social Studies
  • Physical Education
  • Spanish

Ordered List

  1. Math
  2. Science
  3. Social Studies
  4. Physical Education
  5. Spanish

Tables

Tables can also be generated through HTML, using the <table> tag. In order to make a table, three tags needs to be defined. The first one is the <tr> tag. This tag defines each of the table rows. The <th> tag defines the table header, or usually the first row of the table. Finally, the <td> tag defines all the elements inside of the table.

Blocks

The block element is not an actual element that changes the page. The block element is an element that is most commonly used for "enhancements," because of its ability to classify different parts into "blocks," hence its name. The blocks allow you to change its properties very simply without having to go through every single one of the elements and changing them by hand. In order to separate elements into blocks, you need to use the <div> element.

Examples

<div>Brian</div>
<div>Ko</div>

As you can see above, this code will divide "Brian" from "Ko" as a separate entity. Therefore, we can customize each one even though they are part of the same page.

<div style="color:blue;">Brian</div>
<div"color:red;">Ko</div>

The code above will change "Brian" with blue font, and "Ko" with red font.
